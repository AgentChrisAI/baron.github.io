🧩 Marketing Digital Employees Plan

Framework for building digital employees as role-specialized agents powered by markdown workflows, reusable skills, and structured task execution.

Core Thesis

You are not building a loose pile of marketing tools. You are building an agent workforce operating system.

  • Digital employees = role-specialized agents
  • Workflows = repeatable task chains
  • Tasks = instruction + tools + data
  • Markdown files = portable operating memory / SOP layer
  • ClawHub skills = capability modules plugged into workflows

This architecture is stronger than trying to force everything into one monolithic marketing stack.

The Correct Architecture

1) Agent

Who does the work.

Examples: Research Man, SEO Strategist, Content Repurposer, Outreach Operator, RevOps Analyst.

2) Workflow

The repeatable sequence they follow.

Examples: competitor-research-workflow.md, seo-brief-workflow.md, landing-page-audit-workflow.md, content-repurpose-workflow.md, weekly-pipeline-review-workflow.md.

3) Task

The atomic unit of execution. Each task should define:

  • objective
  • instructions
  • required tools/skills
  • inputs/data
  • expected output
  • handoff conditions
  • quality bar

4) Skill

The reusable capability module.

Examples: site scraping, repo analysis, summarization, SEO extraction, social publishing, competitor monitoring, CRM sync, analytics reporting.

What ClawHub Is Actually For

Do not think in terms of “one full marketing suite skill.” Think in terms of small composable skills that slot cleanly into workflows.

The ideal ClawHub skill is:

  • narrow enough to be reliable
  • broad enough to be worth reusing
  • predictable in output
  • easy to reference in markdown workflows

Workforce > Apps

Do not ask: “What tools make up my marketing stack?”

Ask instead:

  • What employees exist?
  • What workflows do they run?
  • What tasks are inside those workflows?
  • What skills power each task?

That is the right abstraction for an agentic company.

Recommended Marketing Workforce

Research Man

Owns market scans, competitor tracking, source extraction, and repo/product research.

Needs skills: tavily, summarize, github, blogwatcher, scraping skills, competitor monitoring skills.

SEO Strategist

Owns keyword research, SERP analysis, content briefs, internal linking plans, and refresh recommendations.

Needs skills: SEO Content Engine, AI SEO, scraping/search/research skills, future custom SERP clustering.

Content Operator

Owns draft generation, content transformation, article packaging, and repurposing.

Needs skills: summarize, content repurpose skills, publishing skills, obsidian.

Social Distribution Operator

Owns channel formatting, scheduling, queue management, and performance feedback loops.

Needs skills: social publishing skills, analytics skills, brand-monitoring skills.

Lifecycle / CRM Operator

Owns lead capture, segmentation, nurture flow tasks, CRM updates, and follow-up triggers.

Needs skills: CRM skills, analytics/reporting skills, email/outreach skills.

Analytics Operator

Owns reporting, attribution summaries, weekly scorecards, and campaign review loops.

Needs skills: analytics skills, reporting skills, data extraction skills.

Suggested Markdown Workflow Schema

# Workflow: Competitor Research Brief

## Purpose
Produce a decision-grade competitor brief on a target company/category.

## Trigger
- New competitor detected
- Quarterly strategy review
- Offer repositioning session

## Inputs
- competitor name
- website
- product category
- key questions
- prior notes

## Required Skills
- tavily
- summarize
- github
- competitor-monitoring
- site-scraper

## Tasks

### Task 1 — Source Collection
**Objective:** Gather primary sources.
**Instructions:** Find official website, pricing, docs, product pages, case studies, and GitHub repos if relevant.
**Output:** Source list with URLs.

### Task 2 — Positioning Extraction
**Objective:** Identify messaging and ICP.
**Instructions:** Extract headline, offer, pain points, claims, proof points, and CTA structure.
**Output:** Positioning summary.

### Task 3 — Capability Comparison
**Objective:** Compare against our offer.
**Instructions:** Identify feature overlap, differentiation, pricing, and obvious weaknesses.
**Output:** Comparison grid.

### Task 4 — Strategic Assessment
**Objective:** Turn findings into action.
**Instructions:** Recommend how we should respond: ignore, match, counter-position, or borrow.
**Output:** Strategic recommendation.

## Deliverable
- Bottom line
- Key findings
- Risks
- Sources
- Recommended action

## Quality Bar
- Primary sources first
- No uncited claims
- Separate facts from inference

Practical Build Sequence

  1. Define the employee roster
  2. Define the top workflows
  3. Map each workflow to required skills
  4. Mark which skills are native, ClawHub, or custom
  5. Build custom skills only where leverage is highest

Three Assets to Build

  1. Agent cards — who each employee is
  2. Workflow markdown library — how each employee works
  3. Skill registry — what capabilities exist and where they come from

Blunt Truth

If you skip the workflow layer and just install skills, you will get capability sprawl, unclear ownership, duplicated effort, and agents that can do things but do not know when or how to do them.

If you anchor the system in markdown workflows first, ClawHub becomes a force multiplier.

Recommended Next Moves

  2. Define the top 10 workflows
  3. Research ClawHub skills by workflow requirement, not by generic category
