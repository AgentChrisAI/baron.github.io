🗂️ Obsidian Synthetic ICP Schema

A practical markdown-first schema for storing evidence-backed synthetic ICPs in Obsidian, designed to support research agents, MiroFish simulations, and CMO-level decision-making.

Purpose

The synthetic ICP should be the canonical structured customer model in your system.

It should not be a fluffy paragraph. It should be a layered, evidence-backed artifact that:

  • captures segment reality
  • captures buyer-role dynamics
  • records triggers, objections, and proof demands
  • stores confidence and provenance
  • feeds simulations, workflows, and strategic decisions

Design Principles

  • Markdown first — all primary artifacts are readable and editable in Obsidian
  • Human readable — no overengineered schema prison
  • Structured enough for agents — predictable sections and light frontmatter
  • Evidence-backed — every important claim should trace to sources
  • Segmented — store multiple ICP variants, not one fake universal customer
  • Simulation-ready — easy to turn into MiroFish seed packets

Core Note Types

1. ICP Segment Note

One note per ICP hypothesis.

Example files:

  • ICP - Core RevOps.md
  • ICP - Expansion Founder Led.md
  • ICP - Contrarian Operator.md

2. Buyer Role Note

One note per role inside the buying system.

Examples:

  • Champion
  • Economic Buyer
  • Technical Evaluator
  • Skeptic
  • Influencer

3. Evidence Note

One note per evidence cluster or source packet.

Examples:

  • Interview Cluster - RevOps Pain.md
  • Competitor Corpus - Apollo.md
  • Reddit Language - SDR automation skepticism.md

4. Simulation Packet

A handoff artifact used to seed MiroFish or other synthetic testing systems.

Linking Model

The notes should form a graph:

  • ICP notes link to buyer-role notes
  • buyer-role notes link to evidence notes
  • simulation packets link to ICPs, roles, and evidence
  • CMO strategy notes link back to simulation packets and ICPs

This keeps the vault human-readable and agent-usable.

Confidence and Provenance Rules

Every ICP note should answer:

  • what is known?
  • what is inferred?
  • what is speculative?
  • what sources support this?
  • how fresh is the evidence?

Do not allow synthetic personas to drift away from evidence without marking it clearly.

Recommended ICP Set

At minimum, maintain:

  • Core ICP — most likely current buyer
  • Expansion ICP — adjacent segment worth testing
  • Contrarian ICP — plausible on paper but risky in reality

This gives you a healthier comparison set than a single sacred ICP.

Workflow Usage

Research workflow

Evidence notes update ICP notes.

Simulation workflow

ICP notes + evidence notes produce simulation packets.

CMO workflow

Simulation packets + evidence notes feed strategic decisions.

Content workflow

Message sensitivities and objections feed copy, landing pages, sales enablement, and nurture flows.

Blunt Rule

The ICP note is the truth layer. The simulation is the challenge layer.

If you let the simulation become the truth layer, the whole system starts hallucinating strategy.
