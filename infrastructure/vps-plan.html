OpenClaw VPS Migration

Created 2026-02-27  ·  Digital Ocean + Mac Node Pairing
🏗 Target Architecture
Digital Ocean Droplet — OpenClaw home base (always-on, isolated from your Mac)
Mac (paired node) — Connected on-demand when your laptop is awake

Ubuntu 22.04 Node.js 22 OpenClaw daemon Telegram / webchat channels Mac node pairing
1
Provision the Droplet
~5 minutes
  • Log in to Digital Ocean → Create → Droplet
  • Choose Ubuntu 22.04 LTS (x64)
  • Size: Basic — $6/mo (1 vCPU, 1GB RAM) — sufficient for OpenClaw
  • Region: closest to Brisbane (Singapore or Sydney)
  • Authentication: SSH key (add your Mac's public key ~/.ssh/id_rsa.pub)
  • Enable Backups (optional but recommended, +$1.20/mo)
  • Create Droplet — note the public IP
2
Initial Server Setup
~10 minutes
  • SSH in: ssh root@<YOUR_IP>
  • Create a non-root user: adduser clawd && usermod -aG sudo clawd
  • Copy SSH key to new user: rsync --archive ~/.ssh /home/clawd/ && chown -R clawd:clawd /home/clawd/.ssh
  • Update packages: apt update && apt upgrade -y
  • Install Node.js 22 via nvm or NodeSource
  • Install npm globally: npm install -g npm
  • Confirm: node -v && npm -v
Note: Install Node.js 22 with: curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - && sudo apt install -y nodejs
3
Install OpenClaw
~5 minutes
  • Switch to clawd user: su - clawd
  • Install OpenClaw globally: npm install -g openclaw
  • Confirm: openclaw --version
  • Run first-time setup: openclaw setup
  • Set your Anthropic API key when prompted
  • Configure your primary channel (Telegram recommended for remote use)
4
Migrate Workspace Files
~10 minutes
  • On your Mac, copy workspace to VPS: scp -r ~/clawd/* clawd@<YOUR_IP>:~/clawd/
  • Exclude node_modules: use rsync --exclude node_modules instead if needed
  • Transfer: SOUL.md, USER.md, MEMORY.md, AGENTS.md, TOOLS.md, memory/, docs/
  • Verify files landed correctly on VPS
  • Transfer any API keys / secrets needed (or re-enter via openclaw config)
Security: Don't SCP your entire ~ — only the clawd workspace. Keep secrets in OpenClaw's encrypted config, not plaintext files.
5
Run OpenClaw as a Service
~5 minutes
  • Start the gateway: openclaw gateway start
  • Verify status: openclaw gateway status
  • Enable auto-start on reboot via systemd (see note below)
  • Test by sending a message via your configured channel
  • Check logs: openclaw gateway logs
Systemd service: Create /etc/systemd/system/openclaw.service with ExecStart=/usr/bin/openclaw gateway start, then systemctl enable openclaw && systemctl start openclaw
6
Pair Your Mac as a Node
~5 minutes
  • On your Mac, install OpenClaw (or ensure it's installed): npm install -g openclaw
  • On the VPS, generate a pairing token: openclaw nodes pair — note the token/URL
  • On your Mac, run: openclaw node join <TOKEN_OR_URL>
  • Approve the pairing from VPS: openclaw nodes approve
  • Verify: openclaw nodes status on VPS shows your Mac as a connected node
  • Test: ask OpenClaw (via chat) to take a Mac screenshot via the node
Note: Your Mac node will show as offline when your laptop is closed/off — that's expected and fine. VPS continues running regardless.
7
Validate & Clean Up
~10 minutes
  • Test all active channels (Telegram, webchat, etc.) — confirm they respond from VPS
  • Test a heartbeat — confirm HEARTBEAT_OK fires on schedule
  • Test memory read/write — confirm MEMORY.md and daily notes persist
  • Test Mac node: camera snap, screen, or file access while Mac is on
  • Once validated, stop OpenClaw on your Mac (don't run two main instances)
  • Update TOOLS.md with VPS IP and SSH details
  • Optional: set up UFW firewall on VPS (ufw allow ssh && ufw enable)
