🐟 MiroFish Analysis

What MiroFish is, how it works, what it appears to cost, what it was built for, and how it can be used for marketing, ICP simulation, and strategy validation.

Bottom Line

MiroFish is a multi-agent simulation and prediction engine. It takes seed material from the real world—news, reports, stories, policies, financial signals—then generates a simulated social world populated by many agents with memory, personas, and interaction logic. It is built to let a user ask “what happens if…” and then inspect a simulation report plus interact with the simulated world.

For marketing and business strategy, the most interesting use is not literal prediction of the future. The real leverage is using it as a high-context scenario simulator for market reactions, ICP behavior, narrative spread, objection emergence, messaging tests, and strategic what-if analysis.

Bluntly: MiroFish looks more useful as a strategic simulation layer than as a normal analytics tool.

What It Does

  • Builds a “parallel digital world” from seed materials and user-defined prediction goals
  • Creates many agents with independent personalities, long-term memory, and behavior logic
  • Runs multi-agent simulation to model how events or narratives might evolve
  • Generates a detailed prediction report
  • Allows interaction with the simulated world and with a report agent after the run

Based on the public README, MiroFish positions itself as a “simple and universal swarm intelligence engine” built to “predict anything.” That positioning is ambitious. The more credible reading is: it is a simulation environment for emergent multi-agent scenario exploration.

How It Works

The public workflow described by the project is:

  1. Graph Building — extract seed information, inject individual and collective memory, and construct a GraphRAG-style knowledge graph
  2. Environment Setup — extract entities and relationships, generate personas, and configure the environment
  3. Simulation — run parallel simulation with dynamic temporal memory updates
  4. Report Generation — a ReportAgent interacts deeply with the post-simulation environment and generates findings
  5. Deep Interaction — the user can talk to simulated agents and the report agent

The README also states the simulation engine is powered by OASIS (Open Agent Social Interaction Simulations), which strongly suggests MiroFish is packaging social multi-agent simulation into a more usable prediction product.

Pricing

No public SaaS pricing was clearly published in the sources reviewed.

What is visible instead:

  • The GitHub repo is public and open source
  • You can self-host it from source or via Docker
  • You need your own LLM API key
  • You need a Zep Cloud API key for memory
  • The README explicitly warns that simulations can have high model consumption and recommends trying fewer than 40 rounds first

So the practical pricing model today appears to be:

  • Software price: open-source / self-hosted
  • Operating cost: LLM inference cost + memory service cost + infrastructure cost

In other words: no obvious flat product pricing, but potentially meaningful runtime cost depending on simulation depth and model choice.

What It Was Built For

Based on the README and public materials, MiroFish was built for:

  • public opinion simulation
  • policy rehearsal
  • financial and macro scenario exploration
  • narrative / story outcome simulation
  • multi-agent “what if” prediction exercises

Its stated vision is broader: a “swarm intelligence mirror” that maps reality and lets users test outcomes before acting. That makes it strategically interesting for business users who want to simulate reactions rather than merely summarize historical data.

Best Use Cases for Marketing and Strategy

This is where it gets interesting.

1) ICP Reaction Simulation

You can use MiroFish as a synthetic market environment where agents approximate customer types, stakeholders, or audience factions. Instead of asking “what does the market think?” you ask “how might different segments react over time if this offer, narrative, or change enters the system?”

Best use: messaging, offer positioning, objection discovery, emotional reaction mapping.

2) Narrative Spread / Reputation Risk

MiroFish appears particularly well suited to simulate public opinion dynamics. That makes it useful for:

  • brand reputation scenarios
  • founder messaging tests
  • product launch backlash simulation
  • PR response rehearsal

3) Product-Market-Fit Hypothesis Testing

You can use it to pressure-test assumptions before shipping a strategy, especially where human reaction and second-order effects matter.

Example questions:

  • How might different buyer personas interpret this positioning?
  • Which objections spread fastest through the simulated market?
  • What happens if we lower price but increase implementation complexity?
  • How might a competitor’s narrative blunt our launch?

4) Market Research Validation Layer

Yes — in principle, you can integrate MiroFish with other research agents and use it as a synthetic validation layer.

The right framing is not “MiroFish is the truth.” The right framing is:

  • research agents gather evidence
  • MiroFish simulates plausible behavior under that evidence
  • you compare simulation outcomes with research conclusions

That makes MiroFish useful as an adversarial or exploratory second pass, especially for testing whether a proposed ICP narrative actually holds up under interaction.

Can MiroFish Act as the ICP?

Yes, but only as a synthetic proxy — not a substitute for real customer evidence.

That means:

  • Good use: simulate buyer reactions, objections, motivations, social spread, internal stakeholder conflict, and scenario divergence
  • Bad use: replacing customer interviews, win/loss data, survey evidence, or actual sales calls

The strongest pattern is:

  1. Research agents build the ICP from real sources
  2. MiroFish instantiates synthetic personas based on that structure
  3. You run scenarios through those personas
  4. You compare simulated reactions against strategic assumptions

So yes, MiroFish can play the role of an interactive synthetic ICP sandbox, which is strategically useful.

How to Get the Best Use Out of It

To get the most value from MiroFish, do not treat it like a black-box oracle. Treat it like a simulation environment that needs strong inputs.

Best practice:

  1. Feed it high-quality seed material: research summaries, customer interviews, objection logs, market reports, competitor analysis, and internal strategy docs
  2. Define a narrow prediction question rather than a vague one
  3. Model specific segments or stakeholder roles, not generic “customers”
  4. Use it to explore divergence, objections, edge cases, and second-order effects
  5. Compare outputs against real evidence, not just intuition

If you put garbage into it, you will get cinematic garbage back.

Marketing and Business Scenarios

Scenario 1 — Offer Testing Before Launch

Goal: Test two offer variants before shipping to market.

Flow:

  1. Research agent compiles ICP pains, buying triggers, objections, and competitor alternatives
  2. MiroFish instantiates segments: skeptical buyer, budget buyer, status-driven buyer, technical evaluator, internal champion
  3. Run simulation on Offer A vs Offer B
  4. Observe which objections dominate, what narratives spread, and which buyer types convert or resist

Strategic value: surfaces failure modes before real spend.

Scenario 2 — Messaging Stress Test

Goal: Find the weak points in positioning.

Flow:

  1. Input homepage copy, product promise, and competitor context
  2. Define target personas and hostile alternatives
  3. Simulate how narratives evolve after exposure to messaging
  4. Extract the phrases that trigger skepticism, confusion, excitement, or dismissal

Strategic value: helps refine copy and angle before publishing.

Scenario 3 — Product-Market-Fit Exploration

Goal: Test whether the current product framing actually aligns with user needs.

Flow:

  1. Feed in customer interview summaries and current product strategy
  2. Build distinct persona clusters
  3. Run adoption, resistance, and advocacy scenarios
  4. Look for mismatch between internal assumptions and simulated behavior patterns

Strategic value: reveals where positioning or product design may be off.

Scenario 4 — Research Validation Layer

Goal: Validate findings from other market research agents.

Flow:

  1. Research agents create a structured market thesis
  2. MiroFish consumes that as seed material
  3. Simulate customer and stakeholder reactions under multiple conditions
  4. Compare whether the thesis survives contact with a more dynamic social environment

Strategic value: acts as a challenge function instead of a confirmation machine.

Scenario 5 — PR / Reputation Simulation

Goal: Rehearse response to controversy, policy change, or sensitive public narrative.

Flow:

  1. Input the event, public context, likely factions, and response options
  2. Run simulations on different response strategies
  3. See how sentiment and narrative spread differ

Strategic value: useful for public-facing companies, founder brands, and sensitive launches.

User Stories

User Story 1 — Founder testing a new narrative

“As a founder, I want to know how three buyer segments react to a new category-defining pitch, so I can refine the message before running paid acquisition.”

How they use MiroFish: input research + draft pitch + segment descriptions, run simulation, extract objections and traction patterns, refine positioning.

User Story 2 — PMM validating product-market-fit assumptions

“As a product marketer, I want to test whether our current value prop resonates with economic buyers, operators, and technical evaluators, so I can adjust positioning before launch.”

How they use MiroFish: convert interview notes and market research into seed material, simulate reaction across stakeholders, identify misalignment points.

User Story 3 — Research team validating an ICP model

“As a research lead, I want to compare our evidence-based ICP with a synthetic behavior simulation, so I can see whether our assumptions break under interaction.”

How they use MiroFish: instantiate synthetic ICPs from research outputs, run scenario prompts, compare simulation to real-world evidence and revise the ICP model.

User Story 4 — Agency or internal team rehearsing a launch

“As a marketing team, we want to pressure-test a campaign before launch, so we can identify backlash, confusion, or weak hooks before spending money.”

How they use MiroFish: feed launch narrative, audience assumptions, competitor context, and risk scenarios; use results to improve campaign structure and response planning.

Integration Pattern with Other Market Research Agents

This is the architecture I would recommend if you want MiroFish to work with your existing agent workforce:

  1. Research agents collect primary sources, interviews, competitor evidence, market signals, and objections
  2. Synthesis agent converts the findings into a structured ICP and scenario packet
  3. MiroFish consumes that packet as seed material and simulates reaction trajectories
  4. Review agent / CMO agent compares simulation output against evidence and makes strategic decisions

This makes MiroFish an interaction simulator, not the sole source of insight.

Where It Fits Best

  • high-uncertainty messaging decisions
  • launch planning
  • narrative and public-opinion simulation
  • scenario planning
  • PMF hypothesis testing
  • research validation / adversarial testing

Where It Fits Worst

  • exact demand forecasting
  • financial precision forecasting without strong models and data
  • replacing real customer research
  • simple reporting tasks where normal analytics are enough

My Assessment

MiroFish is strategically interesting because it sits between:

  • agent simulation
  • market research
  • scenario planning
  • interactive synthetic audience modeling

Its strongest business use is likely synthetic scenario exploration for strategy and messaging, especially when paired with evidence-gathering research agents. If you use it that way, it could become a very sharp tool for pre-launch reasoning and ICP stress testing.

If you use it as a magical truth machine, you will fool yourself.
