Research Brief: GitHub Spec-Kit

Date: 2026-03-23

Requested by: Master (Christopher)

Commissioned by: Baron Von Claw

What is Spec-Kit?

GitHub's spec-kit is an open-source CLI toolkit for Spec-Driven Development (SDD) — a methodology that flips the script on traditional vibe-coding by making specifications executable. Instead of prompting an AI to build things from scratch every time, you:

  1. Define governing principles (/speckit.constitution)
  2. Write what you want built (/speckit.specify) — focus on what, not how
  3. Clarify ambiguities (/speckit.clarify)
  4. Plan the technical stack (/speckit.plan)
  5. Break into tasks (/speckit.tasks)
  6. Execute (/speckit.implement)

Outputs are structured markdown files (spec.md, plan.md, tasks.md, research.md, contracts/) that AI coding agents (including Claude Code ✅) execute against. Supports extensions (new capabilities) and presets (customise templates/terminology).

Research Questions

Agent 1: Spec-Kit Deep Dive

Goal: Fully understand the repo, methodology, CLI, and extension/preset system

  1. What does the full .specify/ directory structure look like after a real project init?
  2. What do the default templates look like (spec-template.md, plan-template.md, tasks-template.md, constitution.md)?
  3. How do extensions work — what community extensions exist? Is there a Jira/GitHub Issues extension? API integration?
  4. How do presets work — what community presets exist?
  5. How does the branching/git workflow integrate? (branches per feature, sequential numbering)
  6. What's in spec-driven.md — the full methodology doc?
  7. How does Claude Code specifically integrate? What slash commands does it expose?
  8. Can spec-kit be used for non-code projects (e.g., n8n workflow JSON, Vapi voice agent configs)?
  9. What does research.md look like in a generated project — is this useful for our agent research pattern?
  10. What are the limitations — what does it not do well?

Agent 2: Application Analysis — How Does This Apply to Us?

Goal: Map spec-kit's value to Master's specific ventures and workflows

Context on Master's world:

  • CopilotHQ (day job): AI consulting/dev firm — builds automations + agents for clients
  • AnyoneAI (platform): Automating the full n8n agency workflow — from spec → build → deploy. Long-game platform play.

  • Primary stack: n8n (automation), Vapi (voice AI), Claude API, Node.js, Python, Supabase, Airtable

Questions to answer:

  1. AnyoneAI fit: AnyoneAI wants to automate the n8n agency workflow. Spec-Driven Development is essentially what AnyoneAI is trying to do — can spec-kit be the specification layer of AnyoneAI? How would you use spec-kit to define n8n workflow specs before building them?
  1. Client delivery at CopilotHQ: Can SDD replace or augment the current approach of building automations? Would speckit.specify → speckit.plan → speckit.implement work for n8n workflow delivery? What would a spec for an n8n automation look like?
  1. Non-code artifacts: n8n workflows are JSON. Vapi assistants are JSON config. Can spec-kit manage specs for these? Is there precedent for non-code deliverables?
  1. Preset/extension opportunity: Could Master build a custom n8n preset for spec-kit that translates SDD into n8n workflow specs, error-handling standards, node naming conventions, etc.? What about a Vapi voice agent preset?
  1. ProntoCall: Could spec-kit help structure the Vapi agent design process — constituting principles for voice agent behaviour, then specifying conversation flows?
  1. Research agent pattern: The spec-kit methodology includes a research.md output and parallel research sub-agents. This mirrors how Baron Von Claw already works. Is there an opportunity to formalise this pattern using spec-kit?
  1. AnyoneAI as a spec-kit wrapper: Could AnyoneAI literally wrap spec-kit — taking client requirements in, running them through the SDD pipeline, and outputting production-ready n8n workflow JSON? What would that product look like?
  1. Competitive/market angle: Is SDD/spec-kit gaining traction? Are other AI agencies using this? Could SunburntAI differentiate by being a "Spec-Driven AI Agency"?
